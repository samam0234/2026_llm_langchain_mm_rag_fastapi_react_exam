대표 코드 예시 — dotenv + 첫 API 호출

import os
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()
api_key = os.getenv("OPEN_API_KEY")
model = os.getenv("OPEN_AI_MODEL")

client = OpenAI(api_key=api_key)

response = client.chat.completions.create(
    model=model,
    messages=[
        {"role": "system", "content": "당신은 AI CCTV 보안 분석 시스템입니다. ..."},
        {"role": "user", "content": "새벽 2시에 창고 출입구에서 사람 2명이 탐지됐습니다."}
    ],
    max_tokens=100
)

answer = response.choices[0].message.content
print(f'GPT 답변 : {answer}')
print(f'토큰 사용량: {response.usage.total_tokens}')

주요 변수 정리

변수 의미 왜 중요한가?
api_key os.getenv("OPEN_API_KEY") 보안 핵심. 코드에 직접 넣지 않음
client OpenAI(api_key=...) 공식 클라이언트 객체
response chat.completions.create 결과 choices, usage, finish_reason 포함
usage response.usage prompt/completion/total 토큰

이 파트에서 배워야 할 핵심 원리

  • → System Prompt로 AI의 역할을 명확히 정의하면 출력 품질이 크게 올라간다.
  • → temperature와 max_tokens를 조절하며 실험 → 창의성 vs 일관성, 비용 trade-off 이해.
  • → JSON 파싱과 finish_reason 확인 → 실제 서비스에서 LLM 응답을 안정적으로 처리하는 방법.
  • → 비용 계산 로직까지 코드로 구현 → 나중에 대시보드나 모니터링에 그대로 사용 가능.
