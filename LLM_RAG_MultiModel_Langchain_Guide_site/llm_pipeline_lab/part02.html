PART 02 • LANGCHAIN

LangChain으로
파이프라인 만들기

LCEL | RunnableLambda | Memory | Output Parser | Agent
복잡한 LLM 애플리케이션을 모듈화하고 재사용 가능하게 만드는 핵심 프레임워크.

LCEL (LangChain Expression Language)
| 연산자로 컴포넌트를 연결. 선언적이고 읽기 쉬운 파이프라인.
RunnableLambda
임의의 Python 함수를 LangChain 파이프라인에 자연스럽게 삽입.
Memory
대화 이력을 저장하고 다음 턴에 전달. 상태를 가진 챗봇의 핵심.
Agent + ReAct
LLM이 도구를 스스로 선택하고 실행하는 에이전트 패턴.

대표 예시 — LCEL 파이프라인 (CCTV 분석)

formatter = RunnableLambda(format_detections)
cctv_prompt = ChatPromptTemplate.from_messages([...])
llm = ChatOpenAI(model="gpt-4o", temperature=0)
json_parser = JsonOutputParser()

analysis_chain = formatter | cctv_prompt | llm | json_parser

result = analysis_chain.invoke(frame_data)
# result → {"risk_level": "주의", "reason": "...", "action": "..."}

Memory를 활용한 실제 챗봇 구조

InMemoryChatMessageHistory에 이전 HumanMessage + AIMessage를 저장 → 다음 호출 시 전체 history를 함께 전달하여 맥락 유지.
self.history.add_user_message(user_msg)
self.history.add_ai_message(ai_response)
messages = [SystemMessage(...), *self.history.messages, HumanMessage(...)]

이 파트의 핵심 메시지

LangChain은 단순히 LLM을 호출하는 라이브러리가 아닙니다.
파이프라인을 조립하고, 상태를 관리하고, 출력을 구조화하는 방법을 배우는 파트입니다.
이 구조를 이해해야 Part 03(RAG)와 Part 04(Multimodal)를 제대로 확장할 수 있습니다.
