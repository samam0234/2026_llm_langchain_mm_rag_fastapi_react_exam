PART 03 • RAG

RAG로 지식을
검색하고 증강하기

ChromaDB + Retriever + Multi-Query
LLM이 모르는 최신/특정 지식을 검색해서 답변에 활용하는 가장 강력한 기법.

Vector Database
ChromaDB에 detection_logs.csv를 임베딩하여 저장
Retriever
사용자 질문과 유사한 과거 탐지 기록을 자동 검색
Multi-Query Retriever
질문을 여러 형태로 변형하여 검색 정확도 향상

RAG가 왜 필요한가?

일반 LLM은 학습 시점 이후의 데이터나 내부 문서를 모릅니다.
RAG는 검색 → 컨텍스트 추가 → 생성 과정을 통해 환각을 줄이고, 최신/특정 도메인 지식을 반영한 답변을 만들어냅니다.

이 파트의 실습 포인트
• detection_logs.csv를 벡터화하여 Chroma에 저장
• cosine similarity로 가장 관련 있는 과거 사건 검색
• 검색 결과를 프롬프트에 주입하여 더 정확한 위험도 판단
